OpenCode integrates with VS Code and terminal-based editors, but the dedicated VS Code extension now has a real **KAIROS** lane instead of acting like a thin terminal launcher.

---

## What is in the extension

- **KAIROS Sidebar** inside VS Code
- **Mode Selector**: `Architect`, `Code`, `Debug`, `Ask`
- **Model Selector** with OpenRouter and OpenCode lanes
- **Proactive reviews** on file save
- **BUDDY score** for lightweight progress feedback
- **Simone actions** for symbol and reference analysis
- **Persistent session steering** instead of spawning a fresh process for every prompt

---

## Install

### Marketplace / normal install

Search for **opencode** in the VS Code Extensions marketplace and install it.

### Manual VSIX install

Download the latest `.vsix` from the [OpenSIN-Code releases](https://github.com/OpenSIN-AI/OpenSIN-Code/releases) page.

Then in VS Code:

1. Open the command palette
2. Run `Extensions: Install from VSIX...`
3. Select the downloaded file

## Usage

### Terminal lane

- `Cmd+Esc` / `Ctrl+Esc` opens or focuses OpenCode
- `Cmd+Shift+Esc` / `Ctrl+Shift+Esc` opens a fresh terminal session
- `Cmd+Alt+K` / `Ctrl+Alt+K` inserts the active file reference

### KAIROS sidebar lane

The sidebar is the deeper integration path.

It keeps a **real local session** alive and routes prompts through the running OpenCode agent instead of launching a new one-shot subprocess for each request.

This gives you:

- persistent context per workspace
- mode-aware behavior
- model switching without throwing away the session lane
- better editor-side continuity

---

## Simone context lane

Inside the sidebar you can trigger:

- **Simone Symbol**
- **Simone Refs**

And in `Architect` and `Debug` mode the extension can inject active-symbol context automatically so KAIROS has a better understanding of what you are pointing at.

---

## ACP vs Session API

The extension is moving toward a deeper ACP-native transport model.

Current direction:

- ACP-first bridge for the sidebar session lane
- persistent local session reuse
- editor-driven save/focus integrations
- session/message transport without a fresh CLI run for every prompt

That makes the VS Code integration much closer to a real agent workspace and much less like a wrapped terminal.

## Troubleshooting

- Make sure `opencode` is installed and available in `PATH`
- If the sidebar cannot talk to the local agent, first start a local OpenCode session in a terminal
- If model selection fails, verify your configured provider/model lanes in OpenCode
- If permissions block tool calls, confirm the prompt in VS Code when KAIROS asks for approval
